🫧 클래스 내부 구성 요소

필드와 지역 변수의 구분
필드(field)는 '클래스에 포함된 변수', 객체의 속성값을 지정할 수 있다. 
지역 변수는 '메서드에 포함된 변수'. 
즉, 필드와 지역 변수는 어떤 중괄호 안에 선언됐는지에 따라 구분할 수 있다.

필드는 힙 메모리의 객체 내부, 지역 변수는 스택 메모리에 생성. 
스택 메모리에 저장되는 변수는 때가 되면 자바 가상 머신이 자동으로 삭제하지만, 
힙 메모리의 객체 안에 저장되는 필드는 객체가 사라지지 않는 한 절대로 삭제 되지 않음

⭐ 1개의 메서드 안에 선언된 모든 지역 변수들의 집합을 '프레임(frame)'이라고 함.

class A{
	int m = 3;	// 상위 중괄호가 클래스일 때는 필드
	int n = 4;	// 필드
	
	void work1(){
		int k = 5;	// 상위 중괄하고 매서드 일때는 지역 변수
		System.out.print(k);
		
		work2(3);
	}
	
	void work2(int i){
		int j = 4;	// 지역 변수
		System.out.print(i+j);
	}
}

반면 지역변수는 직접 초기화하지 않으면 저장 공간이 빈 공간 그대로 있어

🫧 여러 리턴 타입의 메서드 살펴보기
1. 리턴 타입이 void이고 입력 매개변수가 없을 때
리턴 타입이 void일 때는 메서드가 반환하는 값이 없다는 것을 의미. 
즉, 매서드 종료 된 이후에 아무런 값으로 변하지 않으며 
메서드 내부에 'return 리턴값'의 코드가 들어 있지 않다는 것.

void print() {
	System.out.print("hello");
}

2. 리턴 타입이 int이고 입력매개변수는 없을 때
리턴 타입이 int이고 입력매개변수는 없을 때 
메서드 내용의 마지막에는 'return 리턴값'이 포함돼야 한다.

int data(){
	return 3;
}

3. 리턴 타입도 void가 아니고 입력매개변수 포함
double sum(int a, double b){
	return a + b;
}

⭐ 리턴 타입이 void일 때 return 키워드의 의미
void 리턴 타입이라는 것은 리턴하지 않는다는 것을 의미하지만, 
내부에 return 키워드를 사용할 수는 있다. 

이때 return은 '메서드를 종료하라'는 의미다.

void rpintMonth(int m){
	if(1<=m || m<=12) {
		System.out.print("잘못된 입력!");
		return;
	}
	System.out.print(m+"월");
}

' 같은 멤버끼리는 클래스 내부에서 얼마든지 객체를 생성하지 않고 서로를 호출할 수 있다'

배열과 같은 참조 자료형이 입력매개변수로 넘겨질 때 실제 객체가 전달 되는 것이 아니라 객체의 위칫값이 전달

🫧 기본생성자의 자동 추가
생성자가 호출되는 것만으로 객체가 내부적으로 생성, 
생성자의 중괄호 안은 객체가 생성된 이후 할 일이 작성되는 부분이며 

일반적으로 여기서 필드를 초기화.

🫧 내부 객체 참조 변수명인 this 키워드
모든 메서드에는 자신이 포함된 클래스의 객체를 가리키는 this.라는 참조 변수가 있으며
this.를 생략하면 컴파일러가 자동으로 this.를 추가해 주기 때문에 
클래스 내부에서 필드와 메서드를 그대로 사용할 수 있었으며 
지역변수는 멤버가 아니므로 this.가 자동으로 붙지 않음

🫧 클래스 내 다른 생성자를 호출하는 this() 메서드
this()메서드는 자신이 속한 클래스 내부의 다른 생성자를 호출하는 명령이며 
생상자 내부에서만 사용할 수 있고 생성자의 첫 줄에 위치해야한다.

🫧 클래스 외부 구성 요소

* 기호를 사용해 임포트하면 하위 폴더는 힘포지 되지 않으며, 클래스 파일들만 임포트 된다.
